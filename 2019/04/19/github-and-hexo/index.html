Github & Hexo

之前就有使用Git的習慣，所以知道Github可以存放靜態網頁，但到今天才真正的去了解，除了伺服器端的東西無法以外，剩下的例如 HTML JS CSS 等等的都可以在上面執行，那這樣就非常方便了，接著又找到有許多的工具可以幫助你更快的建立相關的版面，於是今天就使用了Hexo建立了現在這個Blog，那麼下面就簡單的紀錄怎麼建立一個GitPage的blog。

Required Package

  • git
  • nodejs
  • npm

Install Hexo & Hexo Git

$ npm install hexo-cli -g
$ npm install hexo-deployer-git --save

Create new Github repository

首先我們必須去Github建立一個新的repository，但這個名稱就有限定，必須以yourName.github.io這樣的規則命名，這樣才可以使用gitPage。

Remote repository setting

$ vim _config.yml
# _config.yml
deploy:
  type: git
  repository: http://github.com/t0239184/alan.github.io.git
  branch: master

Generate static files

$ hexo generate
or
$ hexo g

Theme Setting

Hexo已經有集合了很多人建立好的Theme，可以去他們的網站裡面挑選一個自己喜歡的版型。

Hexo主題 - https://hexo.io/themes/

我們這次使用 NexT 這個主題，所以我們先從 Github 下載主題到 themes 資料夾中，然後之後再稍作修改成自己想要的樣式。

NexT使用文件 - http://theme-next.iissnan.com/

$ git clone https://github.com/iissnan/hexo-theme-next themes/next
$ vim _config.yml
# _config.yml
theme: next

然後更多的設定就需要到 /blog/themes/next/_config.yml下面去設定了，
那麼接下來就要進去將一些頁面顯示的東西打開，例如分類、標籤、關於我、所有文章等等

$ vim themes/next/_config.yml
# themes/next/_config.yml
# 關於Menu Mapping設定
menu:
  home: / || home
  about: /about/ || user
  tags: /tags/ || tags
  categories: /categories/ || th
  archives: /archives/ || archive
  #schedule: /schedule/ || calendar
  #sitemap: /sitemap.xml || sitemap
  #commonweal: /404/ || heartbeat

...
...
  
# 關於側邊欄的位置設定以及顯示方式
sidebar:
  # Sidebar Position, available value: left | right (only for Pisces | Gemini).
  position: left
  #position: right

  # Sidebar Display, available value (only for Muse | Mist):
  #  - post    expand on posts automatically. Default.
  #  - always  expand for all pages automatically
  #  - hide    expand only when click on the sidebar toggle icon.
  #  - remove  Totally remove sidebar including sidebar toggle.
  #display: post
  display: always
  #display: hide
  #display: remove
